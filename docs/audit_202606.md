## 2. Infrastructure Changes

### 2.1 Auto-Merge Workflow Cycle

**Problem:** The `auto-merge-pr.yml` workflow created a circular sync loop.

The workflow:
1. Received a push to `develop`
2. Created `develop-signed` with SSH-signed commits via `git filter-branch`
3. Opened a PR from `develop-signed` → `main`
4. PR merged to `main` (signed commits, different hashes)
5. `develop` fell behind `main` (unsigned originals had different hashes)
6. Manual sync pushed again → triggered step 1 → infinite loop

**Root cause:** Signed commits on `main` and unsigned originals on `develop` have identical content (tree objects) but different commit hashes (signatures change the commit object). After each merge, `develop` was always 1-4 commits behind.

### 2.2 Fix: `sync-develop.yml` (New Workflow)

Added `.github/workflows/sync-develop.yml` — a workflow that triggers when any PR merges to `main` and fast-forwards `develop`:

```yaml
on:
  pull_request:
    types: [closed]
    branches: [main]
```

Key decisions:
- Uses `--force-with-lease` (not plain `--force`) to protect against races
- Checks if `develop` is already synced before pushing (idempotent)
- Pushes via `GITHUB_TOKEN` — no SSH key exposure
- Early-exit guard prevents unnecessary force-pushes

### 2.3 Workflow: `auto-merge-pr.yml` (Restored with Fix)

Original signing workflow restored with an additional sync step at the start:

1. **Sync step:** If `develop` is behind `main`, fast-forward before signing
2. **Sign step:** Create signed commits on `develop-signed` via SSH + `filter-branch`
3. **PR step:** Open PR from `develop-signed` → `main`

The sync step at the beginning ensures that if the previous PR already merged, `develop` gets updated before attempting to sign new commits.

## 3. Code Fixes — Phase 1 (Critical)

### 3.1 `--format` Validation (`scan.go`)

**Before:** Silently fell back to `human` for invalid formats.
```go
if format == "json" {
    // json
} else {
    // human — even for --format jason
}
```

**After:** Explicit validation with error.
```go
switch format {
case "json":
    // json
case "human":
    // human
default:
    return NewExitError(ExitCodeInputError, "Unsupported format '%s'. Use 'json' or 'human'.", format)
}
```

### 3.2 JSON Output Trigger (`scan.go`)

**Before:** `--format human -o file.json` silently generated JSON.
```go
if format == "json" || scanOutput != "" {
```

**After:** JSON only generated when explicitly requested.
```go
if format == "json" {
```

### 3.3 Output File Permissions (`scan.go`, `export.go`)

**Before:** `0644` (world-readable).
```go
os.WriteFile(path, data, 0644)
```

**After:** `0600` (owner-only).
```go
os.WriteFile(path, data, 0600)
```

### 3.4 JSON Output Newline (`scan.go`)

**Before:** Extra `\n` via `fmt.Println`.
```go
fmt.Println(string(jsonBytes))
```

**After:** Clean output via `fmt.Print`.
```go
fmt.Print(string(jsonBytes))
```

### 3.5 Aikido Security Severity (`ci.yml`)

**Before:** Only CRITICAL vulnerabilities blocked CI.
```yaml
minimum-severity: 'CRITICAL'
```

**After:** HIGH and above block CI.
```yaml
minimum-severity: 'HIGH'
```

## 4. Phase 2 — Architecture & Testability

### 4.1 Package-Level Variables → `cmd.Flags().GetX()` (`scan.go`)
Replaced direct use of package-level `format` and `outputPath` variables with `cmd.Flags().GetString("format")` and `cmd.Flags().GetString("output")`. This eliminates shared mutable state and makes the command functions self-contained.

### 4.2 `JSONOutput` Dependency Injection (`output/json.go`)
`JSONOutput` now accepts `io.Writer` instead of hardcoding `os.Stderr`. The constructor `NewJSONOutput(w io.Writer)` enables testability via in-memory buffers.

### 4.3 XML Integration Test (`tests/integration_test.go`)
Added `TestExportXMLFromJSON` — end-to-end test that runs the CLI binary as a subprocess and validates XML output matches the expected schema.

### 4.4 `.golangci.yml` Configuration
Added linter configuration with checks enabled: `errcheck`, `gosimple`, `govet`, `ineffassign`, `staticcheck`, `unused`.

## 5. Phase 3 — Quality & Robustness

### 5.1 Semver Validation (`export.go`)
Added `schemaVersion` validation in the export command. Only major versions 1-2 are accepted; unsupported versions produce a clear warning before failing.

### 5.2 I/O Error Tests (`tests/integration_test.go`)
Added tests for:
- `TestExportInvalidInputFile` — non-existent input file
- `TestExportInvalidJSON` — malformed JSON in input file
- `TestExportUnsupportedFormat` — invalid format argument

### 5.3 Mock TCP Server for Scan Tests (`tests/integration_test.go`)
Replaced dependency on real open ports (`127.0.0.1:22,80`) with a mock TCP server listening on `127.0.0.1:0`. The server accepts and closes connections to simulate an open port without external dependencies.

### 5.4 Windows Signal Handling (`signals.go`)
`SIGTERM` is now only registered on non-Windows platforms (build tag guard). Windows does not support `SIGTERM` in the same way, and registering it caused runtime issues.
