## 2. Infrastructure Changes

### 2.1 Auto-Merge Workflow Cycle

**Problem:** The `auto-merge-pr.yml` workflow created a circular sync loop.

The workflow:
1. Received a push to `develop`
2. Created `develop-signed` with SSH-signed commits via `git filter-branch`
3. Opened a PR from `develop-signed` â†’ `main`
4. PR merged to `main` (signed commits, different hashes)
5. `develop` fell behind `main` (unsigned originals had different hashes)
6. Manual sync pushed again â†’ triggered step 1 â†’ infinite loop

**Root cause:** Signed commits on `main` and unsigned originals on `develop` have identical content (tree objects) but different commit hashes (signatures change the commit object). After each merge, `develop` was always 1-4 commits behind.

### 2.2 Fix: `sync-develop.yml` (New Workflow)

Added `.github/workflows/sync-develop.yml` â€” a workflow that triggers when any PR merges to `main` and fast-forwards `develop`:

```yaml
on:
  pull_request:
    types: [closed]
    branches: [main]
```

Key decisions:
- Uses `--force-with-lease` (not plain `--force`) to protect against races
- Checks if `develop` is already synced before pushing (idempotent)
- Pushes via `GITHUB_TOKEN` â€” no SSH key exposure
- Early-exit guard prevents unnecessary force-pushes

### 2.3 Workflow: `auto-merge-pr.yml` (Restored with Fix)

Original signing workflow restored with an additional sync step at the start:

1. **Sync step:** If `develop` is behind `main`, fast-forward before signing
2. **Sign step:** Create signed commits on `develop-signed` via SSH + `filter-branch`
3. **PR step:** Open PR from `develop-signed` â†’ `main`

The sync step at the beginning ensures that if the previous PR already merged, `develop` gets updated before attempting to sign new commits.

## 3. Code Fixes â€” Phase 1 (Critical)

### 3.1 `--format` Validation (`scan.go`)

**Before:** Silently fell back to `human` for invalid formats.
```go
if format == "json" {
    // json
} else {
    // human â€” even for --format jason
}
```

**After:** Explicit validation with error.
```go
switch format {
case "json":
    // json
case "human":
    // human
default:
    return NewExitError(ExitCodeInputError, "Unsupported format '%s'. Use 'json' or 'human'.", format)
}
```

### 3.2 JSON Output Trigger (`scan.go`)

**Before:** `--format human -o file.json` silently generated JSON.
```go
if format == "json" || scanOutput != "" {
```

**After:** JSON only generated when explicitly requested.
```go
if format == "json" {
```

### 3.3 Output File Permissions (`scan.go`, `export.go`)

**Before:** `0644` (world-readable).
```go
os.WriteFile(path, data, 0644)
```

**After:** `0600` (owner-only).
```go
os.WriteFile(path, data, 0600)
```

### 3.4 JSON Output Newline (`scan.go`)

**Before:** Extra `\n` via `fmt.Println`.
```go
fmt.Println(string(jsonBytes))
```

**After:** Clean output via `fmt.Print`.
```go
fmt.Print(string(jsonBytes))
```

### 3.5 Aikido Security Severity (`ci.yml`)

**Before:** Only CRITICAL vulnerabilities blocked CI.
```yaml
minimum-severity: 'CRITICAL'
```

**After:** HIGH and above block CI.
```yaml
minimum-severity: 'HIGH'
```
